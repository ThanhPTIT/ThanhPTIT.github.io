Tạo blog cá nhân với Hexo & Github Pages trong "1 nốt nhạc"

2021-03-10

Với các CMS phổ biến hiện nay như: wordpress, ghost, joomla… việc tạo một trang blog cá nhân trở nên đơn giản. Tuy nhiên các CMS này khá cồng kềnh trong việc cài đặt do đó sẽ khiến bạn mất một thời gian để làm quen và tìm hiểu. Ngoài ra bạn cũng phải học cách sử dụng một số plugin và themes để đảm bảo trang web được hiển thị một cách tốt nhất. Hexo sẽ giúp giải quyết các khó khăn đó.

Hexo - Blog Framework

Hexo là một blog framework mạnh mẽ hoạt động theo cơ chế Static Site Generator (SSG) giúp tạo ra các static web một cách đơn giản và nhanh chóng. Bạn sẽ không cần bỏ ra quá nhiều thời gian để cài đặt và bảo trì, chỉ cần chạy các Command-line và tận hưởng thành quả. Bạn có thể viết bài bằng Markdown hoặc các markup languages khác. Hexo hỗ trợ rất nhiều plugins và themes được tạo ra bởi các lập trình viên. Hexo tự động generate ra các static file nên thật dễ dàng để deploy lên GitHub Pages, Heroku hay các nền tảng khác.

Hexo is a fast, simple & powerful blog framework.

Github Pages

Github Pages là dịch vụ lưu trữ static web trực tiếp thông qua Github. Static web được tạo ra bởi Hexo sẽ được push lên Github Pages và được truy cập bằng đường dẫn dưới định dạng <github-username>.github.io. Github Pages là một giải pháp tuyệt vời giúp nhanh chóng deploy một trang web hoàn chỉnh khi mà bạn không có host và domain. Ngoài Github Pages bạn có thể lựa chọn các platform khác như: Heroku, Google Cloud, Firebase…

Tạo Github Pages

Việc tạo Github Pages khá đơn giản. Trước tiên bạn phải có 1 tài khoản Github. Sau đó tạo mới một repository với định dạng <github-username>.github.io. Tại file _config.yml, chỉnh sửa lại thông tin deploy.

Tại bước này bạn có thể truy cập vào trang <github-username>.github.io. Tất nhiên chúng ta sẽ nhận được một lỗi 404 Not Found vì repository hiện tại chưa chứa bất kỳ static web nào cả.

Nhiệm vụ tiếp theo là generate các static file và deploy chúng.

Deploy to Github Pages

Thêm một thư mục mới được tạo ra là /.deploy_git. Đây chính là thư mục chứa remote origin của repository mà chúng ta đã tạo. Tại bước này bạn đã có thể xem blog của mình bằng cách truy cập vào trang <github-username>.github.io.

Quá trình deploy thường sẽ mất một khoảng thời gian nhỏ nên đừng lo lắng khi thấy blog của bạn không hiển thị ngay lập tức.

Sau tất cả các bước trên chúng ta đã có một trang cá nhân hoàn chỉnh. Từ đây mỗi khi muốn cập nhật blog của mình, bạn chỉ cần chạy Command-line.

Tổng Kết

Như vậy chỉ với các Command-line, trong thời gian ngắn chúng ra đã có thể tạo ra một trang web hoàn chỉnh. Hy vọng bài viết này có ích với bạn. Nếu gặp khó khăn trong quá trình cài đặt hãy liên hệ Wolf để được giúp đỡ.
